API Documentation

Endpoint 1: Return a list of ALL movies

Description: This endpoint returns a list of all movies available.

Endpoint URL: /movies

Method Type: GET

Query Parameters: None

Request Body: None

Response Data Format: Plain Text

Example Response: "Return a list of ALL movies"

Endpoint 2: Return data about a single movie by title

Description: This endpoint returns data about a movie based on its title.

Endpoint URL: /movies/:title

Method Type: GET

Query Parameters: None

Request Body: None

Response Data Format: Plain Text

Example Response: "Return data about the movie with title: {title}"

Endpoint 3: Return data about a genre by name

Description: This endpoint returns data about a genre based on its name/title.

Endpoint URL: /genres/:name

Method Type: GET

Query Parameters: None

Request Body: None

Response Data Format: Plain Text

Example Response: "Return data about the genre with name: {name}"

Endpoint 4: Return data about a director by name

Description: This endpoint returns data about a director based on their name.

Endpoint URL: /directors/:name

Method Type: GET

Query Parameters: None

Request Body: None

Response Data Format: Plain Text

Example Response: "Return data about the director with name: {name}"

Endpoint 5: Allow new users to register

Description: This endpoint allows new users to register.

Endpoint URL: /users/register

Method Type: POST

Query Parameters: None

Request Body: JSON

Request Body Example:

    {
        "username": "newuser",
        "email": "newuser@example.com",
        "password": "password123"
    }

Response Data Format: Plain Text

Example Response: "User registration successful"

Endpoint 6: Allow users to update their user info

Description: This endpoint allows users to update their user information.

Endpoint URL: /users/:userId

Method Type: PUT

Query Parameters: None

Request Body: JSON

Request Body Example:

    {
        "username": "updateduser"
    }

Response Data Format: Plain Text

Example Response: "Update user info for user with ID: {userId}"

Endpoint 7: Allow users to add a movie to their list of favorites

Description: This endpoint allows users to add a movie to their list of favorites.

Endpoint URL: /users/:userId/favorites

Method Type: POST

Query Parameters: None

Request Body: None

Response Data Format: Plain Text

Example Response: "Movie added to favorites for user with ID: {userId}"

Endpoint 8: Allow users to remove a movie from their list of favorites

Description: This endpoint allows users to remove a movie from their list of favorites.

Endpoint URL: /users/:userId/favorites/:movieId

Method Type: DELETE

Query Parameters: None

Request Body: None

Response Data Format: Plain Text

Example Response: "Movie removed from favorites for user with ID: {userId}, Movie ID: {movieId}"

Endpoint 9: Allow existing users to deregister

Description: This endpoint allows existing users to deregister.

Endpoint URL: /users/:userId

Method Type: DELETE

Query Parameters: None

Request Body: None

Response Data Format: Plain Text

Example Response: "User with ID {userId} has been deregistered"
